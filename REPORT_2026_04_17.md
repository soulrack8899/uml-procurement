## 2. Technical Fixes (Stability & Data Integrity)

### A. Database Infrastructure Modernization
*   **Problem:** Undomesticated schema changes were causing `UndefinedColumn` errors during procurement submissions.
*   **Resolution:** Implemented a **Hard Schema Sync** on startup. The application now auto-detects and injects missing columns (`rejection_reason`, `ledger_url`, `vendor_id`) before accepting connections.
*   **Self-Healing Startup:** Built a startup sequence in `server/main.py` that validates table existence across the dual-database architecture (Auth & Procurement).

### B. Backend Reliability
*   **Resolution for 500 Errors:** Integrated global exception handlers that ensure CORS headers are maintained even during unhandled crashes, improving diagnostic visibility for the frontend.
*   **Null-Safety in Dashboards:** Refactored statistics aggregation logic to gracefully handle null spending values and missing secondary company contexts.
